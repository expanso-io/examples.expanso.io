## Pipeline

| Stage   | Responsibility                                                                  |
| ------- | ------------------------------------------------------------------------------- |
| Gateway | Poll and decode the site-specific OT protocol, then emit delimited records.     |
| Parse   | Map the adapter fields to the example JSON schema and apply configured scaling. |
| Select  | Retain records whose adapter status is nonzero.                                 |
| Label   | Map the adapter status to a neutral review class.                               |
| Route   | Send selected events to Kafka and optionally retain a local copy.               |

The gateway or adapter is required. The example socket input is a line receiver; it is not a Modbus, DNP3, OPC UA, or IEC 61850 client.

## Limitations and assumptions

<Limitations>

- The sample records are synthetic and already decoded.
- Register maps, scaling factors, status meanings, and data classifications require site review.
- Kafka and local-file delivery are configured but have not been exercised by repository evidence.
- No protocol integration, end-to-end execution, operating envelope, delivery behavior, or deployment posture has been assessed.

</Limitations>
