## Pipeline

| Stage           | Reference behavior                                                                           |
| --------------- | -------------------------------------------------------------------------------------------- |
| Collect + parse | Tail selected files and map authored JSON, syslog, or CEF-like samples into an event schema. |
| Filter          | Delete or sample records according to an explicit policy.                                    |
| Enrich          | Add configured source, host, and routing fields.                                             |
| Deliver         | Wrap retained records for a Splunk HEC HTTP request.                                         |

This independent technical reference does not assert a vendor relationship or feature parity with Splunk tooling. Source filtering permanently changes what reaches the destination.

## Limitations and assumptions

<Limitations>

- The repository has not executed the full file-to-HEC path.
- The sample parser does not cover every valid syslog, CEF, or multiline format.
- Retention, security, audit, and investigation requirements must determine what can be filtered.
- Ingest, license, storage, and query outcomes depend on the Splunk product, contract, and workload; none are measured here.

</Limitations>
