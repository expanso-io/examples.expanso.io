## Pipeline

| Stage        | Reference behavior                                                                           |
| ------------ | -------------------------------------------------------------------------------------------- |
| Prepare      | Generate synthetic POS events, add static store metadata, and apply the example field rules. |
| Batch + ship | Group records, configure Parquet encoding, and write to S3-compatible storage.               |
| Query        | Show external SQL for registering and inspecting the objects.                                |

## Limitations and assumptions

<Limitations>

- No POS connector is included, and the repository has not executed the full Expanso, S3, and MotherDuck path.
- File discovery and table registration behavior depend on the selected analytics service and account configuration.
- Batch size, compression, freshness, and query behavior are unmeasured; synthetic transactions do not establish data coverage or quality.

</Limitations>
