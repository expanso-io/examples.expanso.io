## Pipeline

| Stage    | Responsibility                                                    |
| -------- | ----------------------------------------------------------------- |
| Call     | Invoke the configured downstream processor with bounded settings. |
| Catch    | Handle a processor error and annotate the record.                 |
| Fallback | Select another destination or a local output path.                |
| Inspect  | Review the resulting record and branch metadata.                  |

The stage sequence explains configuration changes; it is not a runtime availability test.

## Limitations and assumptions

<Limitations>

- The request records and failure states are authored examples.
- No current machine gate starts the HTTP service, database, or fallback destinations.
- Retry timing, duplicate delivery, recovery, backpressure, and destination durability are not assessed.
- Tune timeouts and retry counts against the behavior of each real dependency.

</Limitations>
