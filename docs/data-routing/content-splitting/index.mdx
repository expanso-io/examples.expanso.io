## Pipeline

| Stage          | Responsibility                                             |
| -------------- | ---------------------------------------------------------- |
| Select         | Locate the collection that should become separate records. |
| Split          | Emit an output message for each selected element.          |
| Copy context   | Retain identifiers needed by downstream processing.        |
| Validate shape | Route or reject records that lack expected fields.         |

The Explorer compares authored before-and-after records; no maintained fixture currently executes the configuration.

## Limitations and assumptions

<Limitations>

- All payloads and resulting records are synthetic.
- Ordering, partial failure, large-message limits, memory pressure, and downstream acknowledgements are not assessed.
- Copied context may duplicate sensitive fields; review field selection before adapting the pattern.
- Nested and delimited inputs need format-specific validation.

</Limitations>
