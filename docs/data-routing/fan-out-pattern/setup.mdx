# Fan-Out Configuration Review

Kafka, object storage, and search are external integrations. The repository does not execute this multi-destination topology end to end.

<ExampleSurface
  kind="reference"
  title="Before you adapt it"
  description="Resolve the example boundary before changing processors or destinations."
>

- An Expanso environment for configuration inspection
- Reachable destination services and environment-managed credentials
- Destination-specific topics, buckets, indexes, schemas, and access rules

</ExampleSurface>

1. [Broker branch](/data-routing/fan-out-pattern/step-1-configure-broker)
2. [Kafka branch](/data-routing/fan-out-pattern/step-2-add-kafka)

## Adaptation checks

- Decide whether branch failures are isolated or propagated.
- Test retries, duplicates, replay, and partial destination outages.
- Set retention and access controls separately for every destination.

This route is a configuration review, not a verified execution guide.
