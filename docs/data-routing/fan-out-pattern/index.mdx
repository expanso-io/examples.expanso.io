## Pipeline

| Stage          | Responsibility                               |
| -------------- | -------------------------------------------- |
| Broker         | Add the first configured output branch.      |
| Kafka          | Attach a separate streaming destination.     |
| Object storage | Add a batch-oriented storage branch.         |
| Search         | Add an indexing branch with its own mapping. |

A successful branch does not prove that another branch accepted or retained the record.

## Limitations and assumptions

<Limitations>

- The reference destinations require services, credentials, schemas, and access configuration.
- No current gate executes the full multi-destination topology.
- Cross-branch ordering, retry isolation, duplicate delivery, replay, and partial failure are not assessed.
- Retention and access rules belong to each destination environment.

</Limitations>
