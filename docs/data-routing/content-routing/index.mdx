## Pipeline

| Stage      | Responsibility                                         |
| ---------- | ------------------------------------------------------ |
| Severity   | Assign a route from the event severity.                |
| Region     | Apply a region-specific branch before general routing. |
| Event type | Select a destination from the event category.          |
| Priority   | Combine fields into an explicit priority decision.     |

Branch precedence is part of the configuration. The example does not infer policy or validate a destination.

## Limitations and assumptions

<Limitations>

- The input events and destination labels are synthetic.
- Region fields are routing inputs, not evidence that a transfer meets legal or organizational obligations.
- No destination service, credential, delivery retry, or dead-letter behavior is exercised by a current machine gate.
- Review branch order when adding overlapping conditions.

</Limitations>
