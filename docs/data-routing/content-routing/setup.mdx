# Content Routing Configuration Review

This pattern evaluates synthetic fields and sends records to named branches. Destination availability and organizational policy are outside the checked-in example.

<ExampleSurface
  kind="reference"
  title="Before you adapt it"
  description="Resolve the example boundary before changing processors or destinations."

- An Expanso environment for configuration inspection
- A documented input schema for every routed field
- Real destination outputs if you adapt the example beyond local review

</ExampleSurface>

## Adaptation checks

- Order overlapping conditions explicitly.
- Define a default branch for unexpected field values.
- Validate destination schemas and failure handling in your environment.

This route is a configuration review, not a verified execution guide.
