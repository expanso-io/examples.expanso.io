## Pipeline

| Stage          | Responsibility                                 |
| -------------- | ---------------------------------------------- |
| Severity       | Map severity values to a starting priority.    |
| Customer tier  | Adjust the authored score from a second field. |
| Combined score | Evaluate multiple criteria in a defined order. |
| Age adjustment | Demonstrate an authored escalation rule.       |

This pattern labels and routes records; it does not implement a scheduler or prove processing fairness.

## Limitations and assumptions

<Limitations>

- Priorities, tiers, weights, and age thresholds are illustrative.
- The example does not execute a queue broker or worker pool.
- Starvation, service levels, capacity, admission control, and delivery behavior are not assessed.
- Review scoring rules for bias and unintended precedence before adaptation.

</Limitations>
