## Pipeline

| Stage          | Responsibility                                     |
| -------------- | -------------------------------------------------- |
| Classify       | Assign a priority class from record fields.        |
| Score          | Calculate a numeric routing value.                 |
| Output         | Choose configuration for each priority class.      |
| Age adjustment | Demonstrate an escalation field for older records. |

The visual stages explain record transformation rather than simulating a durable offline queue.

## Limitations and assumptions

<Limitations>

- Backlog records and connection conditions are authored examples.
- The current machine gate does not exercise disconnection, restart, disk exhaustion, or replay.
- Durability, ordering, capacity, duplicate delivery, and starvation behavior are not assessed.
- A real deployment needs bounded storage and explicit overload handling.

</Limitations>
