# Troubleshoot the DB2 to BigQuery Reference

| Symptom                       | Check                                                                                 |
| ----------------------------- | ------------------------------------------------------------------------------------- |
| DB2 connection fails          | Driver, DSN, network path, TLS, credentials, and query permissions.                   |
| Mapping rejects a row         | Required fields, input types, date formats, currency code, and null behavior.         |
| Account fields are unexpected | Mask format, deterministic-key input, salt or key ownership, and downstream joins.    |
| BigQuery rejects output       | Dataset, table schema, field types, partition field, credentials, and write mode.     |
| Duplicate or missing rows     | Source watermark, retries, idempotency key, partial writes, and reconciliation query. |

Use synthetic or approved test records and a non-production destination while diagnosing. The repository has not exercised these integrations.

**Reference:** [Open the combined configuration](/enterprise-migration/db2-to-bigquery/complete-pipeline).
