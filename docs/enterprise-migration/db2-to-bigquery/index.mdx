## Pipeline

| Stage     | Reference behavior                                                                  |
| --------- | ----------------------------------------------------------------------------------- |
| Extract   | Read selected columns through a DB2-compatible SQL driver.                          |
| Annotate  | Add source and pipeline fields for downstream review.                               |
| Transform | Apply authored currency rates, field reduction, category mapping, and schema names. |
| Validate  | Reject records missing fields required by the reference.                            |
| Deliver   | Write transformed records through a BigQuery output.                                |

The mapping is illustrative; it does not translate DataStage jobs automatically. Source queries, rate data, tokenization, schema contracts, reject handling, and destination idempotency remain implementation decisions.

## Limitations and assumptions

<Limitations>

- The repository has not executed this configuration against DB2 or BigQuery.
- Rates and categories are authored constants; the account transformation establishes neither anonymization nor adherence to any regulatory standard.
- Scheduling, retries, schema migration, reconciliation, and rollback are outside the reference.

</Limitations>
