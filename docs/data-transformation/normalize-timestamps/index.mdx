# Normalize Timestamps

Build a production-ready timestamp normalization pipeline at the edge.

## The Problem

Events arrive with timestamps in inconsistent formats and time zones:
- ISO8601 with timezone offsets, Unix epochs, custom formats
- Different timezones make analytics impossible
- No consistent time basis for comparison
- GDPR Article 30 requires timezone audit trails

## The Solution

Learn 3 timestamp processing techniques:

1. **Parse Multiple Formats** - Format detection with Bloblang parsing for ISO8601, Unix epochs, custom patterns
2. **Convert Timezones** - Timezone detection and UTC conversion while preserving original context
3. **Enrich Metadata** - Extract year, month, hour, quarter, weekend flags for business intelligence
