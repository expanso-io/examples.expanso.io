## Pipeline

| Input      | Intended parser output                           |
| ---------- | ------------------------------------------------ |
| JSON       | Selected fields with normalized names and types. |
| CSV        | Columns mapped by an explicit schema.            |
| Access log | Fields extracted from one declared text pattern. |
| Syslog     | Header, facility, severity, and message fields.  |

Route parse failures separately instead of silently treating them as valid structured events. Preserve enough source context to diagnose a bad parser or changing upstream format.

## Limitations and assumptions

- The Explorer data and expected records are curated; no current machine gate executes the parsers.
- Multiline records, escaping variants, malformed inputs, encoding, truncation, and parser-version differences are not assessed.
- The included patterns are scoped to the shown fixtures.
