## Pipeline

| Stage        | Intended behavior                                                   |
| ------------ | ------------------------------------------------------------------- |
| Tumbling     | Assign an event to one fixed time bucket.                           |
| Sliding      | Reuse events across overlapping lookback windows.                   |
| Session      | Group activity until an inactivity threshold is reached.            |
| Hierarchical | Produce summaries for configured sensor, location, and global keys. |

The diagrams and results are authored examples. They are useful for inspecting field shape and configuration intent, not for inferring measured reduction, load behavior, or event-time correctness.

## Limitations and assumptions

- No maintained machine gate executes these window configurations.
- Late arrival, watermark, clock-skew, cache eviction, restart, and multi-node behavior are not assessed.
- The shown thresholds and aggregates are fixture choices, not recommended operating values.
