## Pipeline

| Strategy      | Key material             | Main tradeoff                                      |
| ------------- | ------------------------ | -------------------------------------------------- |
| Exact content | The serialized event     | Changes in any included field produce a new key.   |
| Fingerprint   | Selected business fields | Field selection defines what counts as equivalent. |
| Identifier    | A supplied event ID      | Correctness depends on the producer's ID contract. |

Cache lifetime and key construction jointly define the comparison window. Keep the original event separate from generated comparison metadata unless downstream consumers explicitly need it.

## Limitations and assumptions

- The cache examples have no maintained execution or restart evidence.
- Serialization order, concurrency, eviction, replay, collision response, and multi-node coordination are not assessed.
- Hashing creates comparison keys; it is not a privacy control for retained source fields.
