## Pipeline

| Stage   | Intended responsibility                                                |
| ------- | ---------------------------------------------------------------------- |
| Parse   | Decode the submitted JSON and reject the configured invalid shape.     |
| Enrich  | Add processing and correlation fields.                                 |
| Select  | Apply the configured severity score and filter.                        |
| Redact  | Delete selected fields and replace configured identifiers with hashes. |
| Fan out | Configure Kafka, S3, and local backup outputs.                         |

Hashed identifiers remain pseudonymous and may still be personal or sensitive data. Destination-specific access, retention, encryption, and deletion controls remain external to this example.

## Limitations and assumptions

- The HTTP input, mappings, destinations, retries, and fan-out behavior are not executed by a current machine gate.
- Kafka and S3 credentials, schemas, delivery semantics, replay, backpressure, recovery, and operating limits are not assessed.
- The architecture does not establish privacy, security, legal, or operational readiness.
