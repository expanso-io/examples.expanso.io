## Pipeline

| Stage   | Intended change                                       |
| ------- | ----------------------------------------------------- |
| Enrich  | Add configured node, pipeline, and processing fields. |
| Reshape | Separate event data from processing metadata.         |
| Batch   | Group records by configured count or period.          |
| Export  | Write a local copy and configure S3 object delivery.  |

Metadata fields are useful when their source, clock, and update semantics are known. Object keys, retention, encryption, access, and replay policy belong to the destination design.

## Limitations and assumptions

- AWS credentials, bucket policy, network access, retries, and object delivery are not exercised.
- Batch loss, duplicate objects, partial writes, restart behavior, and backpressure are not assessed.
- The synthetic records and displayed objects are curated examples.
