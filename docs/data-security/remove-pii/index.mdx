## Pipeline

| Stage        | Intended change                                               |
| ------------ | ------------------------------------------------------------- |
| Payment data | Delete the full card number and expiry                        |
| IP address   | Replace the source value with a salted hash                   |
| Email        | Extract the domain, add a salted hash, and delete the address |
| User name    | Replace the name with a shortened salted hash                 |
| Location     | Delete latitude and longitude                                 |

The remaining fields can still identify or help link a person when combined with other data. Treat field selection, salts, retention, access, and downstream joins as separate design decisions.

## Limitations and assumptions

The repository machine gate runs the canonical HTTP-to-file configuration with one synthetic fixture, fixed test salts, a pinned Benthos image, and exact expected output. The Explorer stages are still curated. This narrow check does not establish production readiness, broader data coverage, privacy assurances, or an operating envelope.
