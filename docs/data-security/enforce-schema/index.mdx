## Pipeline

| Stage    | Responsibility                                      |
| -------- | --------------------------------------------------- |
| Define   | Describe required fields and value constraints.     |
| Validate | Apply the configured schema processor.              |
| Route    | Select accepted or rejected output from the result. |
| Adapt    | Extend the authored schema and failure metadata.    |

Validation checks the declared shape; it does not prove semantic correctness or destination durability.

## Limitations and assumptions

<Limitations>

- Schemas and records are synthetic fixtures.
- No current machine gate executes the complete accepted and rejected paths.
- Schema evolution, compatibility, replay, rejected-record access, and remediation are not assessed.
- A matching shape can still contain incorrect or unsafe values.

</Limitations>
