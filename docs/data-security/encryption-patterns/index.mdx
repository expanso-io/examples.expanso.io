## Pipeline

| Stage    | Responsibility                                                 |
| -------- | -------------------------------------------------------------- |
| Payment  | Separate retained display fields from protected values.        |
| Identity | Apply encryption to selected identity fields.                  |
| Address  | Protect street-level values while retaining chosen geography.  |
| Dates    | Protect source dates and derive explicitly configured cohorts. |

The stages are design patterns, not a comparative cryptographic evaluation.

## Limitations and assumptions

<Limitations>

- All records and keys are synthetic example inputs.
- No maintained fixture currently executes the full reference pipeline.
- The configuration does not include a key service, authorization model, rotation procedure, or recovery test.
- Derived clear fields can remain sensitive and need independent review.

</Limitations>
