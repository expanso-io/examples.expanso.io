## Pipeline

| Stage        | Responsibility                                          |
| ------------ | ------------------------------------------------------- |
| Tag          | Add origin metadata used by later routing.              |
| Minimize     | Remove fields not needed by the example output.         |
| Pseudonymize | Replace selected identifiers with derived values.       |
| Generalize   | Reduce precision for selected location and time fields. |

Field transformation reduces exposed detail but does not determine identifiability or legal treatment.

## Limitations and assumptions

<Limitations>

- The records, regions, identities, and transfer destination are synthetic.
- Pseudonymized records can remain linkable to people when combined with salts, source data, or other attributes.
- The example does not assess transfer mechanisms, purpose, retention, access, deletion, or re-identification risk.
- Obtain privacy and legal review for the complete system and data flow.

</Limitations>
